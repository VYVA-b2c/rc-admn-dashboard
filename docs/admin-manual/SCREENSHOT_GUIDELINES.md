# Screenshot Guidelines

Screenshots are part of the product documentation and must be safe to share.

## Data Rules

- Use sanitized demo data only.
- Do not show real names, phone numbers, emails, addresses, medical details, notes, IDs, or tokens.
- Replace live client names with neutral examples such as `Demo Client`.
- Replace staff names with neutral examples such as `Red Cross Staff`.
- Replace emails with `admin@example.org`.
- Replace phone numbers with `+00 000 000 000`.

## Capture Rules

- Capture the final loaded UI, not skeleton/loading states.
- Capture only the useful part of the screen when a full-page image makes details unreadable.
- For admin-only dialogs, use a safe demo tenant. If live write controls cannot be shown safely, use a sanitized UI example and label it as such.
- Avoid toast errors, browser chrome, dev tools, cursor focus rings, and temporary local-only banners.
- Keep screenshots visually consistent: same viewport, same theme, same organization style.

## Review Rules

Before publishing:

- Check every screenshot at readable zoom.
- Confirm no loading skeletons are visible.
- Confirm no real data is visible.
- Confirm the caption matches the screenshot.
